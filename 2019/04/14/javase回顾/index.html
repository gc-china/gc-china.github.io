面向对象与面向面向过程区别

面向过程

优点：性能比面向对象高。因为类调用需要实例化，资源消耗大

缺点 没有面向对象的易复用，易扩扩展

面向对象

优点 易维护 易扩展 易维护 封装多态继承特性 可以低耦合

缺点 性能比面向过程低

JDK JRE JVM 区别

JVM

Java虚拟机（JVM）是运行 Java 字节码的虚拟机。JVM有针对不同系统的特定实现（Windows，Linux，macOS），目的是使用相同的字节码，它们都会给出相同的结果 是java 跨平台的关键 一处编译多处运行

JRE

JRE 是 Java运行时环境。它是运行已编译 Java 程序所需的所有内容的集合，包括 Java虚拟机（JVM），Java类库，java命令和其他的一些基础构件。但是，它不能用于创建新程序

JDK

DK是Java Development Kit，它是功能齐全的Java SDK。它拥有JRE所拥有的一切，还有编译器（javac）和工具（如javadoc和jdb）。它能够创建和编译程序

重载和重写的区别

  • 重载： 在一个类中 方法名相同，参数类型不同，参数个数不同，顺序不同，方法返回值和访问修饰符可以不同 在编译期
  • 重写： 发生在字类和父类中 方法名 参数列表必须相同 访问修饰符必须大于等于父类如果父类方法修饰符为private 子类不能重写该方法

Java 米面向对象 三大特性：封装继承多态

  • 封装:

    封装把一个对象私有化，同时提供一些可以被外界访问属性方法(get set)

  • 继承

    继承是面向对象的一个重要特性，通过继承，子类可以使用父类中的一些成员变量方法，从而能够提高代码的复用性，提高开发的效率。被继承的类叫做基类或者父类继承基类或者父类的类叫派生或子类。继承是通过extends 关键字来实现的 使用格式为 class子类名 extends 父类名

    • 子类只能有父类非private 的方法和属性
    • 子类可以重写父类的非私有化方法
    • 子类可以有自己的属性和方法
    • java 是单继承的 所以子类最多一个父类可以多个接口实现

  • 多态

    一种定义多种实现 就是多态 多态分为两种

  • 运行时多态
  • 编译时多态

String StringBuffer StringBuider 的基本区别

  1. Stringprivate　final　char　value[] 是用Final 修饰的 所以String 对象是不可变得 而StringBuilder 与 StringBuffer 都继承自 AbstractStringBuilder 类，在 AbstractStringBuilder 中也是使用字符数组保存字符串char[]value 但是没有用 final 关键字修饰，所以这两种对象都是可变的
  2. String 对象是不可变得,线程安全 StringBuffer 对方法加了同步锁是线程安全的 StringBuilder 并没有对方法同步 是线程不安全
  3. StringBuilder >Stringbuffer>String 在性能方面

自动装箱与拆箱

装箱：将基本类型用他们相对应应用类型包装起来

拆箱：将包装类转为基本数据类型

接口和抽象类的区别

  1. 接口方法默认是public 所以方法必须是抽象的 抽象类可以有非抽象的方法
  2. 接口定义的变量默认是final类型 抽象类不一定
  3. 一个类可以实现多个接口，只能继承一个实现类
  4. 一个类实现接口要实现该接口的所以抽象方法，抽象类不需要

main() 方法

  • punlic static void main(String[] args) 是程序的入口方法，JVM 在运行程序时会首先查找main()方法 其中 public 是权限修饰符号 表明任何类和对象可以访问 static表明是一个静态方法 即方法存在方法区 只要类加载后 就可以不用通过实例化对象访问可以直接通过类名.main()方法直接访问从而来查找入口地址若能找到就执行 找不到就报错！

  • 因为 main方法 是程序入口 当程序运行时第一个执行的是main方法 通常来说要执行一个类的方法

    必先实例化对象 因此在编写main方法的时候就应该不用实例化就能调用这方法 所以main方法要public 和static 修饰

  • main 方法的其他可用格式

    • static public void main(String[] args)
    • public static final void main(String[]args)
    • 也可用Synchronized 修饰 static public synchronized void main()
    不管哪种定义哪种必须保证main 方法返回值为void 并有static 和public 因为main 方法为程序入口 所以不能abstract修饰

什么是构照函数以及构照函数的特点

构照函数是一种特别的函数，用来在对象的实例化时初始化对象的成员变量

特点:

  • 构照函数必须与类名相同并且没有返回值
  • 每个类的可以用多个构照函数，如果在创建类的时候没有提供构照 jvm会在编译字节码的过程中会提供一个无参额构照函数 该构照不会执行任何代码
  • 构照可以用0个1个和1个以上的参数
  • 构照函数随着new 操作一起调用，并且不能由编写者自己调用 必须由系统调用且只执行一次
  • 构照函数的主要作用是完成对象的初始化
  • 构照不能被继承，不能被重写，可以重载
  • 子类可以通过supe 调用父类的构照函数

什么是反射

反射是java语言中的一个非常重要的机制，它允许运行时自我检查也允许对其内部类的成员进行操作，因此能增加灵活性！反射提供功能主要是

  • 得到一个对象所属的类
  • 获取类的所以方法和变量
  • 在运行时创建对象
  • 在运行时调用对象

抽象类和接口的异同

概念: 如果一个类中包含抽象方法，那么这个类就是抽象类 在Java中可以通过类或者类中的某些方法声明为abstract(abstract 只能修饰类和方法，不能修饰属性)来表示一个类是抽象类 接口是一个方法的集合，接口的方法都是抽象方法 都没有方法体 在Java中接口是interface 来实现的

抽象类和接口的相同点

  1. 都不能被实例化

  2. 接口的实现类或者抽象类的子类都只要实现了接口的或者抽象类的抽象方法才能被实例化

    接口和抽象类的不同点

    1.在Java8 之前接口中只能定义 其方法不能再接口中实现 只有实现接口的类才能实现接口定义的方法 而抽象类可以直接定义方法和实现！(就是接口中只能含有抽象方法，抽象类可以有自己方法和实现,当一个类有抽象方法时该类一定要声明成抽象类)

  3. 接口需要实现,抽象类只能被继承，Java 是单继承体系 所以可以实现多个接口只能继承一个抽象类可以实现多个借口来达到多重继承的目的！
  4. 接口强调特定功能的实现而抽象类强调所属关系
  5. 接口中定义的成员变量默认为public static final 只能是静态的不能修改的成员变量 必须给其赋初始值,其所有的方法都是public abstract 的修饰并且只能被这这个修饰
  6. 抽象类可以有自己的成员变量和非抽象的成员方法抽象类的成员变量默认为default(仅本包可见)而且可以使用其他的修饰符修饰 也可以重新赋值！抽象类中的抽象方法和不能用private static Synchronized native 等访问符修饰
  7. 接口运用于实现比较常用的方功能便于维护或者添加删除，而抽象类更倾向于充当于公共类，不适合对日后代码的重新修改
