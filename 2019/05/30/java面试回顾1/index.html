Java 面向对象

  1. 面向对象都有那些特性以及你对这些特性的理解

    1. 继承：继承是已有类被称为子类,继承变化中的软件系统有一定的延续性,同时继承也是程序中可变因素重要手段
    2. 封装：通常认为封装是把数据和操作数据的方法绑定起来，对数据的访问只能通过已定义的接口。面向对象的本质就是将现实世界描绘成一系列完全自治、封闭的对象。我们在类中编写的方法就是对实现细节的一种封装；我们编写一个类就是对数据和数据操作的封装。可以说，封装就是隐藏一切可隐藏的东西，只向外界提供最简单的编程接口
    3. 多态： 简单的说就是用同样的对象引用调用同样的方法但是做了不同的事情。多态性分为编译时的多态性和运行时的多态性。如果将对象的方法视为对象向外界提供的服务，那么运行时的多态性可以解释为：当 A 系统访问 B 系统提供的服务时，B 系统有多种提供服务的方式， 但一切对 A 系统来说都是透明的。方法重载（overload）实现的是编译时的多态性（也称为前绑定），而方法重写(override)实现的是运行时的多态性（也称为后绑定）。运行时的多态是面向对象最精髓的东西，要实现多态需要做两件事：1. 方法重写（子类继承父类并重写父类中已有的或抽象的方法）；2. 对象造型（用父类型引用引用子类型对象，这样同样的引用调用同样的方法就会根据子类对象的不同而表现出不同的行为）。
    4. 抽象：抽象是将一类对象的共同特性总结出来构照类的过程,包括数据抽象两方面抽象只关注对象有哪些属性和行为,不关注这些行为细节是什么！

  2. Java中有没有goto语句？

    • goto是Java 中的保留字,在目前版本的Java中没有使用 gotohe const 是两个无法使用的关键字,因此被称为保留字

  3. &和&&的区别

    • &的运算符有两种用法
      • 按位与
      • 逻辑与
    • && 运算符是短路与运算, 如果&& 左边的表达式值为false，右边的表达式会直接短路掉，不会进行运算！例如在验证登入判断用户名是否为空的时候判断用户是否为空而且不是空字符串应当写完 username !=null && username.equals(“”) 二者的顺序不能交换，更不能用&运算符，因为第一个条件如果不成立，根本不能进行字符串的 equals 比较，否则会产生 NullPointerException 异常。注意：逻辑或运算符（|） 和短路或运算符（||）的差别也是如此。

  4. 在Java 中如何跳出当前的多重嵌套循环

    • 在最外层加一个标记A然后用break A 可以跳出多重循环(尽量直接使用break 和commit 这样会使代码阅读性变低)

  5. 两个对象相同(x.equals(y)==true) 他们的hashCode 值是否相等

    • 相等 应为java规定如果两个值的equals 方法返回true 那么HashCode 一定相同
    • equals 比较的是比较对象的内存地 址 和对象的值的Hascode 同时相等 才返回true
    • 与== 相比 == 比较的是值的话 是比较数值是否相等 == 比较的是对象的话 比较的是对象的内存地址

  6. 是否可以继承String

    • String 是final 的 不可以被继承，不可以修改

  7. 重载和重写的区别

    • 方法重载和重写都是多态的一种体现，重载是编译器多态，重写是运行时多态
    • 重载和重写的规则不一样
    • 重载 方法名相同形参列表可以不同，返回值可以不同,修饰符也可以不同，重写参数列表必须和被重写的参数完全一样，返回值和重写的方法也必须一样 重写的方法修饰符不能低于被重写的方法的修饰符

  8. char 的变量能不能储存一个中文汉字

    • char 类型可以存储一个中文汉字，char 在java中占用2个字节 一个中文汉字也是占用2个字节

  9. 抽象类和接口有什么区别

  • 抽象类

    • 抽象类可以定义构照器

      • 抽象类可以有抽象方法和具体方法

  • 接口中的成员都是public 的

    • 抽象类可以定义成员变量
    • 有抽象方法的类必须声明抽象类,抽象类不一定要有抽象方法
    • 抽象类可以包含静态方法

  • 接口

    • 接口不能定义的构照器
    • 方法全部都是抽象方法
    • 抽象类的成员可以是private 默认 protected public
    • 接口不能有经验方法
    • 一个类可以实现多个接口
  • 相同点
    • 不能实例化
    • 可以将抽象类和接口类型作为引用类型
    • 一个类继承了某个抽象类或者实现某个接口都需要对其中抽象方法全部进行实现，否则该类不必声明为抽象类
  1. == 和equals 的区别
    • == 和equals 一个是运算符，一个是方法
    • == 如果比较的是基本数据类型 则比较的值对象是否相等, 如果比较的是引用类类型的话 比较的是对象的内存引用地址是否相同
    • equals 比较的两个对象的内容是否相等
    • equals 不能比较基本数据类型
  2. break 和 continue 的区别
    • break 和continue 都是跳出循环
    • continue 跳出了本次循环执行下一次循环 break 是结束循环，跳出循环体执行循环后的语句
  3. String s=”hello” s=s+”word”； 这两行代码后，原始的String对象中的内容改变了没
    • 没有改变 因为String 是被设计成不变的类 原来的S =”Hello” 但是对 s+”hello” 后s 变成 “helloword 但是s 指定的是另一个另一个String 对象了 “ 但是原来的String 在堆对象中 没有改变！
  4. Java 中实现多态的机制是什么
    • 靠的是父类或者接口的定义的引用变量可以指向子类或者其子类具体的实例对象

Java 异常处理

  1. java 异常分几种

    • 编译时异常
    • 运行时异常

  2. erro 和exeption 的区别

    • Erro 和exeption 都是Throwable 类
    • Erro 是错误 一般与JVM相关的问题 是不可以处理的异常
    • Exception类 表示是程序可以处理的异常，可以捕获处理，可以是程序恢复运行！
    • Execption 分为运行时异常和编译时异常 编译时异常 必须提前处理 不能通不过编译 运行时异常要处程序不会处理 出现这类异常，程序会终止。而受检查的异常，要么用try。。。catch 捕获，要么用throws 字句声明抛出，交给它的父类处理，否则编译不会通过

  3. 请说出最常见的5个RuntimeExcption

    1. 空指针异常：java.lang.NullPointerException
    2. java.lang.ClassNotFoundException 指定的类找不到；出现原因：类的名称和路径加载错误
    3. java.lang.NumberFormatException 字符串转换为数字异常
    4. java.lang.IndexOutOfBoundsException 数组角标越界异常
    5. ava.lang.ClassCastException 数据类型转换异常

  4. Throw 和throws 的区别

    • throw

      1. throw 语句用在方法体内，表示抛出异常
      2. throw 是具体向外抛出异常的动作，所以它抛出的是一个异常实例，执行
        throw 一定是抛出了某种异常

    • throws

      1. throws 语句用在方法声明后面，表示抛出异常 由方法的调用来进行异常处理

      2. throws 表示出现异常的一种可能性，并不一定会发生这种异常

      3. throws 主要是声明这个方法会抛出某种类型的异常，让它的使用者要知道需要捕获的异常的类型

  5. final finally finalize 的区别

    1. final 用于声明属性 方法 类 表示属性不可变，方法不可以覆盖，类不可以继承
    2. finally ：是异常处理的一个阶段 表示不管如何必定执行
    3. finalize ：Object 类的一个方法，在垃圾回收器执行的时候会调用被回收对象的此方法

常用API

  1. 数组有没有length() 方法，String 有没有length() 方法
    • 数组没有length 数组是有length 属性 String 是有length 的方法
    • 数组 length 属性是获取 数据的中元素个数 String 的length() 是或者String的字符长度
  2. String StringBuilder StringBuffer 的区别
    • String 是只读字符串 是final 修饰的 是不可改变的
    • StringBuffer/StringBuilder 表示的字符串对象可以直接进行修改
    • StringBuffer/StringBuilder 他们的方法都一样 但是Stringbuffer 是性能底下 线程安全 StringBuilder 是线程安全的！
